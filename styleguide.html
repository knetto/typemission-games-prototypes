02 / System overview

Identity & rules

TYPEMISSION is a family of spy-mission typing games. Each game uses a dark 16:9 tactical frame in which the player operates one physical machine by typing. The shared system covers the mission header, briefing flow, hardware layers, input states and system report; the center object gives each game its identity.

One mission hueEach mission owns one background hue (red, indigo, amber, purple, grey or green). Panel, border and muted-text roles are derived from that hue.
Semantic accentsOrange #e48e35 means active/action, green means correct/available and red means error/threat. Fingerprint and defense promote their local yellow or green accents where documented.
Panels are machinesEvery panel earns hardware details: rivets, LEDs, ports, signal bars, etched serial text. Nothing looks like a generic website card.
Same flow, alwaysBriefing cards → game arena → in-machine feedback → system report. The player advances with the spacebar at every step.
Glow is meaningA glow is feedback, not decoration: orange = type this now, green = done, red = wrong. Idle parts stay matte.
NeverLight backgrounds (except document paper), rounded pill buttons, blue hyperlinks, generic web inputs, more than one accent glowing at once on the same object.

03 / Extracted from each game's :root

Color system

Every game fills in the same set of color roles with different values. That is the whole theming system: components never hard-code a color, they take it from the game's role palette — panel color, border color, hairline color, screen background, dim text. A new game only needs its own set of values for those roles.

Shared semantic accents

Large swatches show the colors at a useful scale. Click any swatch to copy its exact value.

06 / One shell, three views

Frame & flow

Every game lives inside one fixed 16:9 frame, centered on a near-black page with a matrix-rain canvas behind it. Inside the frame a header stays fixed on top and the stage below it swaps between exactly three views.

Frame recipe: a responsive 16:9 aspect ratio, page-hue background, 3px border in the theme's soft tint, corner radius 12, heavy drop shadow and clipped content. The header sits on top; the stage fills the rest.

Why a set driver pin leans

Light torque on the plug makes the plug and housing bores sit slightly out of alignment. Because real locks have tiny manufacturing tolerances, one pin stack binds first. When its key-pin/driver-pin boundary reaches the shear line, the plug can rotate a fraction and hold the driver above that line; the next stack then becomes the binding pin. The guide exaggerates that trapped state with a small left-or-right lean so it remains legible at game scale.

  1. Pick: lifts the active key pin.
  2. Torque: comes from a tension tool, not from the pick itself.
  3. Set: the driver clears the shear line and catches above it.
  4. Transfer: slight plug rotation makes another stack bind.

12 / Interaction grammar

Feedback & motion

Feedback is instant, local and color-coded. The same five moments exist in every game:

MomentWhat happensMotion
Awaiting inputExactly one thing glows orange: the current slot, key, pin, node, word or virus. Everything else stays matte.Slow breathing pulse on the target (≈1s), gentle 2s text pulse on prompts.
Correct inputThe typed part flips to green (border, text, glow) and stays green. Progress moves to the next target.Brief success flash on the key (~0.4s), packet fired / pin lifted / word revealed.
Wrong inputThe part flashes red, a mistake indicator turns red (lock orb, shield node, mistake stat), terminal logs a danger line.Quick shake ≈0.3s, red flash ≈0.5s, then back to neutral.
CompletionThe whole machine turns green: metal plate border + glow, all inputs green, status label flips (VERGRENDELD → ONTGRENDELD), green overlay with "TOEGANG VERLEEND".Green glitch-flash 0.6s, unlock icon pops in with overshoot, then calm glow.
FailureRed lockout: machine border flashes red, red inset glow, overlay "SYSTEEM VERGRENDELD", then the red-stamped report.Alarm flash ≈0.5s ×3, background flicker, report stamp slams in.
