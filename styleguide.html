01 — System overview

Identity & rules

TYPEMISSION games are spy-mission typing games. Every game is a dark, 16:9 tactical interface in which the player hacks one physical machine by typing. The machine is always built the same way: a light grey metal plate, with a dark device panel bolted on top, holding a CRT screen with scanlines and the input parts the player types on. Section 05 documents that stack layer by layer.

One hue per gameEach game owns exactly one background hue (red, indigo, amber, purple, grey, green). All panel, border and muted-text colors are tints of that hue. See section 02.
Shared accents everywhereOrange #e48e35 = the active target. Green #9ad744 = correct/unlocked. Red #ff3b30 = mistake/threat. These never change per game.
Panels are machinesEvery panel earns hardware details: rivets, LEDs, ports, signal bars, etched serial text. Nothing looks like a generic website card.
Same flow, alwaysBriefing cards → game arena → in-machine feedback → system report. The player advances with the spacebar at every step.
Glow is meaningA glow is feedback, not decoration: orange = type this now, green = done, red = wrong. Idle parts stay matte.
NeverLight backgrounds (except document paper), rounded pill buttons, blue hyperlinks, generic web inputs, more than one accent glowing at once on the same object.

02 — Extracted from each game's :root

Color system

Every game fills in the same set of color roles with different values. That is the whole theming system: components never hard-code a color, they take it from the game's role palette — panel color, border color, hairline color, screen background, dim text. A new game only needs its own set of values for those roles.

Shared accents (identical in every game)

04 — One shell, three views

Frame & flow

Every game lives inside one fixed 16:9 frame, centered on a near-black page with a matrix-rain canvas behind it. Inside the frame a header stays fixed on top and the stage below it swaps between exactly three views.

Frame recipe: 16:9 ratio (80vw × 45vw, capped at 80% of the screen height), page-hue background, 3px border in the theme's soft tint, corner radius 12, heavy drop shadow, content clipped. The header sits on top; the stage fills the rest.

05 — Machine anatomy

The layer stack

Every machine in the door games is the same sandwich. Build it in this order and it will look like the existing games: L1 metal plate → L2 device panel → L3 screen → L4 inputs, with neon cables running behind the device panel. Each layer is shown here on its own first, then assembled.

L1 — Light grey metal plate

The physical carrier. Vertical grey gradient, thick lighter border, four rivets, an etched label, and the three mistake indicators. It never glows — until the mission is complete, when its border turns green.

The plate also carries a small status label under the device panel: red spaced-caps text with a blinking 6px dot while locked; both flip to steady green when the mission completes.

Values: vertical gradient #4a4d52 → #33363b (40%) → #2a2d32 · border 2.5px #5a5e65, radius 8 · inner top highlight (10% white) + inner bottom shading · rivets are 8px radial circles (#6a6e75 → #3a3d42) set 10px from each corner · the label is etched, never glowing: Orbitron 900, 25%-white text, 3px letter-spacing, faint dark chip behind it · the 3 lock circles are the error budget — green (8% green fill) flipping to red (15% red fill) per mistake, with a quick scale "shock" bounce.

L2 — Dark device panel

The hacking rig bolted onto the metal. Near-black (#0d0e12), thin white-ish border, and always framed by hardware: a header strip (rig label + two blinking LEDs) and a footer strip (animated signal bars + port jacks). The middle stays empty at this layer.

L3 — Screen with scanlines

The CRT. Uses the theme's screen background, an orange uppercase label, tiny 7px system tags in the corners, and — always — the scanline overlay with a 0.15s flicker. This is the "TV lines" effect; do not skip it on any screen surface.

09 — One machine per game

Game object library

Each game swaps its own input object into layer L4 of the stack. These are the exact looks, each shown in its own game theme.

Lockpicking — pin chambers

Six vertical chambers inside a dark cylinder housing. Each chamber holds, top to bottom: a silver coil spring, a grey metal driver pin and an orange key pin carrying the letter. A dashed orange shear line crosses the housing as an alignment guide. Typing the letter lifts the pin pair so the seam between driver and key pin meets the shear line; the picked pin turns brighter orange, a fully solved pin turns green, and the spring stays compressed. Pin heights differ per chamber — the lock looks mechanical, not uniform.

10 — Interaction grammar

Feedback & motion

Feedback is instant, local and color-coded. The same five moments exist in every game:

MomentWhat happensMotion
Awaiting inputExactly one thing glows orange: the current slot, key, pin, node, word or virus. Everything else stays matte.Slow breathing pulse on the target (≈1s), gentle 2s text pulse on prompts.
Correct inputThe typed part flips to green (border, text, glow) and stays green. Progress moves to the next target.Brief success flash on the key (~0.4s), packet fired / pin lifted / word revealed.
