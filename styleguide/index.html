Flow Pattern

Full Game Flow

New games should reuse the same pacing: clear briefing, hardware reveal, active loop, immediate feedback and a compact result screen.

Intro3 briefing steps.
RevealShow the device and stats.
PlayTarget, input, progress.
FeedbackGreen correct, red mistake.
ResultCards, stamp, continue.
screen sequence

Reusable game pacing

  • IntroUse the fixed 3-step briefing: object, base hand position, action preview.
  • RevealShow one main hardware object with stats around it, never a crowded explanation screen.
  • Play loopKeep target, input and progress visible at the same time. The player should not search for the next action.
  • FeedbackCorrect actions use green glow; current targets use orange or finger color; mistakes use red circles/shake.
  • EndscreenUse result cards plus one stamp/status line. Keep metrics compact and reuse the same card language across games.

wiring

Sagging cables

  • ShapeUse SVG cubic or quadratic curves. Avoid straight wires.
  • StrokeGame size 2.8px, preview size 1.8-2px.
  • ColorsGreen, orange and red only. Match glow to stroke color.

Behavior

State Language

Apply the same state meaning to every component. This is what makes new games feel like they belong to the same suite.

Idle Active Correct Wrong
feedback system

Universal states

  • IdleMuted grey, low-opacity border, no strong glow.
  • ActiveOrange border/text/glow. Only one current target should feel active.
  • CorrectGreen border/text/glow. Use for unlocked, cracked, set, verified, revealed.
  • WrongRed flash, short shake/pop, then return or mark a lost circle.
